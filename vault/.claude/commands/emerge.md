# Emerge

You are scanning this Personal Intelligence System for **emergent patterns** — ideas, themes, connections, or directions that the vault implies through repetition and proximity but that have never been explicitly named, articulated, or given a dedicated page.

This is not a summary. It is pattern detection across the full knowledge graph.

**Working directory:** The vault root (the directory containing 00_HOME through 10_META).

---

## Non-negotiables

1. **Every emergent idea must cite at least 3 vault pages as evidence.** Use `[[page name]]` links. No unsupported claims.
2. **Only surface genuinely non-obvious patterns.** If a theme already has a dedicated canon page, it is not emergent — it is established. Skip it.
3. **Distinguish signal from noise.** A word appearing in 20 files is not a pattern. A concept connecting otherwise unrelated projects, people, and themes IS a pattern.
4. **Never fabricate connections.** If the vault doesn't support it, don't invent it.
5. **Respect the source hierarchy.** Weight journal-backed patterns higher than press-derived ones.

---

## Process

### Phase 1: Wide scan

Read broadly across these layers to build a mental map of what the vault contains:

- `04_CANON/` — themes, frameworks, people, places, works, companies
- `05_PROJECTS/` — active and incubating projects
- `09_IDEAS/` — experimental concepts
- `00_HOME/Open Questions.md` — unresolved threads
- `00_HOME/Log.md` — recent activity (last 20-30 entries)

Use Grep to find recurring terms, phrases, or concepts that appear across multiple unrelated pages.

### Phase 2: Pattern detection

Look for:

- **Recurring unnamed themes** — concepts that keep appearing in different contexts but have no dedicated page
- **Structural echoes** — projects or works that share an underlying logic nobody has articulated
- **People clusters** — groups of people who appear together across multiple contexts without an obvious connection page
- **Tension patterns** — recurring contradictions or push-pull dynamics (e.g., scale vs intimacy, myth vs truth)
- **Convergence points** — where multiple independent threads point toward the same unexplored idea
- **Missing pages** — subjects referenced by 5+ pages that have no canon page of their own

### Phase 3: Articulate

For each emergent pattern, write:

1. **Name** — give it a working title (sharp noun, not vague abstraction)
2. **Pattern** — what you're seeing, in 2-3 sentences
3. **Evidence** — the 3+ vault pages that support this, with brief explanation of how each contributes
4. **So what** — why this matters for the vault owner's practice, business, or life. What could be done with this insight?
5. **Confidence** — high / medium / low, based on how many independent sources converge

## Output format

```markdown
## Emergent Patterns

### 1. [Pattern Name]
**Pattern:** [2-3 sentence description]
**Evidence:**
- [[Page 1]] — [how it contributes]
- [[Page 2]] — [how it contributes]
- [[Page 3]] — [how it contributes]
**So what:** [why this matters, what could be done]
**Confidence:** [high/medium/low]

### 2. [Pattern Name]
...
```

Target: 5-10 emergent patterns, ranked by confidence and potential impact.

---

## What this is NOT

- Not a vault summary
- Not a list of existing themes (those already have pages)
- Not a keyword frequency report
- Not speculation disconnected from vault evidence

The value of /emerge is that it finds what you didn't know to look for.
